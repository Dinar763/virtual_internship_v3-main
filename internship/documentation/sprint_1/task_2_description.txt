Build проекта

Команда build в Gradle выполняет ряд действий, связанных со сборкой проекта. Вот основные шаги,
которые она включает:

1. Загрузка конфигурации проекта: Gradle загружает файлы конфигурации (например, build.gradle),
считывает настройки и применяет плагины.
2. Сборка проектных зависимостей: Gradle разрешает зависимости, указанные в проекте,
и скачивает необходимые артефакты из репозиториев.
3. Компиляция кода: Выполняется компиляция исходного кода (Java, Kotlin и т.д.) в байт-код,
который затем может быть запущен на Java Virtual Machine (JVM).
4. Тестирование: Запускаются тесты, если они определены в проекте. Это может включать
 юнит-тесты и интеграционные тесты.
5. Создание артефактов: Формируются конечные артефакты, такие как JAR-файлы или WAR-файлы,
в зависимости от типа проекта.
6. Очистка промежуточных файлов: В случае необходимости Gradle удаляет временные файлы
и директории, созданные в процессе сборки.
7. Публикация артефактов: Если проект настроен на публикацию, артефакты могут быть
отправлены в удаленные репозитории.
